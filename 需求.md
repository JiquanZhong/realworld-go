1. 产品简介

MCP 广场（MCP Hub） 是一个开放的、面向开发者与平台用户的「AI 能力扩展市场」。
通过 MCP 广场，用户可以浏览、搜索、安装并使用社区或开发者提供的 MCP 服务，从而扩展自身智能体的能力。

MCP（Model Context Protocol）是一种 标准化的 AI 工具扩展协议，可以让第三方服务以统一接口接入用户的智能体中。

MCP 广场的核心目标：

展示 MCP 服务生态

让用户能简单、一键安装 MCP

扩展平台智能体能力

鼓励开发者贡献 MCP

构建能力生态网络效应

MCP 广场是整个 AI 平台的 能力增长引擎。

2. 产品目标
2.1 业务目标

建立一个可持续扩张的 AI 工具生态

降低用户安装 MCP 的成本

增加智能体的能力数量和使用深度

提升平台用户留存与粘性

吸引第三方开发者来扩展 MCP 系统

2.2 用户目标

找到适合自身场景的 MCP

一键安装并直接使用

查看 Demo 示例

根据口碑（评分）判断 MCP 质量

收藏好用的 MCP

提交自己的 MCP 服务

3. 用户角色与权限
用户	权限描述
未登录用户	浏览 MCP 列表、查看详情、查看评论、搜索筛选
普通用户（已登录）	安装 MCP（生成 token）、调用 MCP、收藏 MCP、评论与评分、提交 MCP

5. 核心功能模块

MCP 广场的功能分为 五大模块：

5.1 MCP 浏览与搜索模块
功能点：

MCP 列表展示（卡片）

搜索（按名称、描述、标签）

筛选（按分类、标签、评分、安装量、更新时间）

排序（热门、评分、最新）

分类（7 大类）：

数据查询

Web API

文件处理

数据库

企业工具

AI 推理 / 模型

地图 / 地理

娱乐

5.2 MCP 详情模块

MCP 详情页应包含：

MCP 名称、图标

分类、标签

描述

Endpoint URL

MCP JSON Schema（可折叠查看）

安装量、评分

Demo（curl、Python、JS 示例）

使用场景

评论区（评分 + 文本）

收藏按钮

5.3 MCP 安装与 Token 管理模块（核心）
安装流程

用户点击“安装 MCP”

后端创建一个 MCP Token：

绑定用户

绑定 MCP

具备有效期

可自动续期

智能体调用 MCP 时自动插入 token

用户无需手动管理 token

Token 格式：
Authorization: Bearer <mcp_token>

调用验证：

MCP 服务端必须验证：

token 是否有效

是否过期

是否具有对应的 MCP 使用权限（scope）

5.4 MCP 评论与评分模块

用户可对 MCP 评分（1–5 星）

用户可填写文字评论

可按评分排序

管理员可删除不合规评论

5.5 MCP 提交模块

用户可提交 MCP 服务供大家使用：

字段包括：

MCP 名称

Endpoint

JSON Schema（自动解析）

分类

标签

描述

Demo 示例（文本形式）

无需审核 → 自动展示
（管理员仍可在后台进行下架操作）

6. 安全与鉴权机制
6.1 用户登录鉴权（JWT Session）

登录用户才能安装和使用 MCP。

未登录 → 只可浏览

6.2 MCP Token 鉴权

Token 由 MCP Hub 后端生成，包括：

token 字符串

过期时间

用户 ID

MCP ID（scope）

示例：

{
  "token": "eyJhbGc...xyz",
  "expire_at": 1739500000,
  "scope": "mcp:weather-api"
}

6.3 调用流程安全性

智能体调用 MCP 时由系统自动插入 token

用户不会暴露 token

第三方 MCP 服务必须验证 token
